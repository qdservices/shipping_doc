    Introduction

    Welcome to the OmniShip Cross Border API Documentation!

    The OmniShip Cross Border API allows you to perform cross border operations on the OmniShip system.

    OmniShip Cross Border API is built using REST principles which ensures predictable URLs that makes writing applications easy. This API follows HTTP rules, enabling a wide range of HTTP clients which can be used to interact with the API.

    Every resource is exposed as a URL. The URL of each resource can be obtained by accessing the API Root Endpoint https://crossborder.omniship.eu/api/v1/.

    For example the Customers resource is reachable at https://crossborder.omniship.eu/api/v1/customers.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "https://crossborder.omniship.eu/api/v1/<resource>"
  -H "Authorization: FkihCtzyXWvutSRUaaEupN8hvABcDefgHI6lJKvv"
  -H "Content-Type: application/json"

    Make sure to replace FkihCtzyXWvutSRUaaEupN8hvABcDefgHI6lJKvv with your API key.

    OMNISHIP uses API keys to allow access to the API. Contact your OMNISHIP representative to receive a new OMNISHIP API key.

    In order to authenticate properly, please put Authorization: Bearer <API Access Token> in the request header. OMNISHIP expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: FkihCtzyXWvutSRUaaEupN8hvABcDefgHI6lJKvv

    You must replace FkihCtzyXWvutSRUaaEupN8hvABcDefgHI6lJKvv with your personal API key.

    A customer is the owner of an uploaded shipment. The customer code is needed to identify the customer on uploading a new Waybill pre alert.

    Attribute Type Description
    id integer Unique ID of a customer. This is used as an identifier.
    code string Unique code to identify a customer.
    name string Name of the customer
    weight_unit string The default weight unit used for a customer on upload.
    created_at string Date when the customer resource was first created.
    updated_at string Date when the customer was last updated.

    Create a Customer

    curl "https://crossborder.omniship.eu/api/v1/customers"
  -X POST
  -H "Authorization: FkihCtzyXWvutSRUaaEupN8hvABcDefgHI6lJKvv"
  -H "Content-Type: application/json"
  -d '{
        "code": "ADS",
        "name": "Amsterdam Delivery Service",
        "weight_unit": "grams"

    A waybill is a document that represents the contents of a shipment ready for transport. It details the exact quantity, price and delivery details of the individual packages. Perform the simple operations mentioned below to create and manage your Waybill pre alert.

    Attribute Type Description
    customer_code string Unique ID of a customer. This is used as an identifier.
    waybill_type string The type of waybill. Valid values are ('air', 'road', 'train', 'sea', 'other').
    waybill_number string Identifies the total shipment. If waybill type is 'air' a validation check is done on the format of the waybill_number given.
    reference string Unique identifier of the shipment. Usually an id of the original system where the pre-alert was uploaded.
    weight_unit string Unit of weight used in the pre-alert. Valid values are ('grams', 'kilograms')
    pre_alert_rows array A pre alert contains multiple pre_alert rows. Each pre alert row contains order_number, package_barcode, bag_grade, shipping_method, weight, company_name, consignee_name, phone_number, email_address, street, house_number, house_number_extension, additional_address_info, city_or_town, state_or_province, zip_code, country_code, sku_code, sku_description, quantity, currency, price, shipping_costs, hs_code, website

    This endpoint creates a new Waybill pre alert in OMNISHIP cross border.

    HTTP Request

    POST https://crossborder.omniship.eu/api/v1/waybills

    Arguments

    Attribute Type Description
    customer_code string required The unique customer code. This code should reflect a customer code in the database.
    waybill_type string required The type of waybill. Valid values are air, road, train, other
    waybill_number string required Unique number of the waybill. This number cannot already exist in the database for the same customer.
    reference string required Unique reference usually the ID of the external system where shipment is uploaded from.
    weight_unit string optional The weight unit used for the weights in the pre-alert. Valid values are grams, kilograms. When omitted or empty the default customer weight unit will be used.
    pre_alert_rows array required Line with details of the specific shipment. Each line must contain order_number, bag_grade, country_code.

    Pre Alert Row Parameters

    Attribute Type Description
    order_number string required Order number for the shipment line.
    package_barcode string optional Barcode of the package. Usually the final mile carrier barcode.
    bag_grade string required Barcode of the bag/box containing the packages.
    shipping_method string optional Method of shipping. Can be used as an identifier of the implementation hub.
    weight number required Weight of the package described on the shipping line. The weight will be reflected using the weight_unit given.
    company_name string optional Name of the company of the package consignee.
    consignee_name string optional Name of the of the package consignee.
    phone_number string optional Phone number of the of the package consignee.
    email_address string optional Email address of the of the package consignee.
    street string optional Street address of the of the package consignee.
    house_number number optional House number of the of the package consignee.
    house_number_extension string optional House number extension of the of the package consignee.
    additional_address_info string optional Additional address info i.e. Apt 10.
    city_or_town string optional City or town of the of the package consignee.
    state_or_province string optional State or province of the of the package consignee.
    zip_code string optional Zip code of the of the package consignee.
    country_code string required Country code of the of the package consignee. (needs to be ISO 3166 2-char code)
    sku_code string optional SKU code of the of the item(s) in the package.
    sku_description string optional Description of the of the item(s) in the package.
    quantity number optional Quantity items in the package.
    currency string optional Currency of the value of the item in the package.
    price number optional Value of the item in the package.
    shipping_costs number optional Shipping costs of the package.
    hs_code string optional HS tariff code for customs.
    website url optional URL of the details page of the specified item.

    Errors

    The Cross Border API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The resource requested is hidden for administrators only.
    404 Not Found -- The specified resource could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The resource requested has been removed from our servers.
    429 Too Many Requests -- You're requesting too many resources.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
